# 飞书应用机器人配置指南（L2：交互命令）

通过飞书应用机器人，在群聊中 `@机器人 /sou` 即可查询余额、查看状态、停止阅读等。

> **前置**：L1（Webhook 推送通知）无需额外配置，只需填入 Webhook URL 即可。  
> **L2** 需要额外在飞书开放平台创建应用，适合需要交互控制的用户。

---

## 🏗 架构

```
飞书群聊 @机器人 /sou
      ↓
飞书服务器 → POST 回调
      ↓
你的服务器 :6700/feishu/callback (feishu-bot.js)
      ↓
解析命令 → 执行（查余额/状态/停止）→ 调用飞书 API 回复消息
      ↓
飞书群聊 ← 回复消息
```

---

## 📋 配置步骤

### 第 1 步：创建飞书应用

1. 打开 [飞书开放平台](https://open.feishu.cn/app)
2. 创建企业自建应用 → 填写名称（如「V2EX Helper」）
3. 进入应用 → **添加应用能力** → 添加「机器人」

### 第 2 步：配置机器人事件

> ⚠️ **关键区分**：飞书控制台有「事件配置」和「回调配置」两个 Tab。  
> 我们需要的是 **「事件配置」**（接收消息事件），不是「回调配置」（卡片交互）。

1. 左侧菜单 → **事件与回调** → **事件配置** Tab（默认即是）
2. 订阅方式选择：**将事件发送至开发者服务器**
3. 点击 **添加事件** → 搜索 `接收消息` → 选择 **`接收消息 v1.0`** → 确认
4. **请求网址** 填入：
   ```
   http://8.135.36.248:6700/feishu/callback
   ```
5. 飞书会发送 URL 验证请求到你的服务器，验证通过后点击保存

> 📖 **为什么是「事件」而不是「回调」？**  
> - **事件**：飞书推送消息给你，你收下即可，无需立即响应 → 适合机器人接收消息  
> - **回调**：飞书等你同步返回内容展示给用户（如卡片按钮点击）→ 本项目不需要

### 第 3 步：获取凭据

在飞书开放平台应用的 **凭证与基础信息** 页面，复制以下值：

| 配置项 | 飞书平台位置 |
|---|---|
| `App ID` | 凭证与基础信息 → App ID |
| `App Secret` | 凭证与基础信息 → App Secret |
| `Verification Token`（可选） | 事件订阅 → Verification Token |

### 第 4 步：填写服务器配置

```bash
# 编辑环境变量文件
nano /root/.v2ex_env
```

添加：
```ini
FEISHU_APP_ID=cli_xxxxxxxxxxxx
FEISHU_APP_SECRET=xxxxxxxxxxxxxxxxxxxxxxxxxx
FEISHU_VERIFICATION_TOKEN=xxxxxxxxxxxxxxxx
FEISHU_BOT_PORT=6700
```

### 第 5 步：安装并启动

```bash
# 安装 systemd 服务
cd ~/v2ex-max-helper
sudo bash scripts/install-systemd.sh --feishu-bot

# 或手动
systemctl start v2ex-feishu-bot
systemctl status v2ex-feishu-bot
```

### 第 6 步：添加机器人到群

1. 飞书开放平台 → 应用「V2EX Helper」→ **发布** → 创建版本并发布
2. 在飞书群中 → 设置 → 群机器人 → 添加机器人 → 搜索「V2EX Helper」

## 🤖 可用命令

| 命令 | 功能 |
|---|---|
| `@V2EX Helper /help` | 显示帮助 |
| `@V2EX Helper /sou` | 查询余额 |
| `@V2EX Helper /status` | 查看运行状态 |
| `@V2EX Helper /debug` | 查看最近日志 |
| `@V2EX Helper /stop` | 停止阅读器 |

---

## 🔒 安全

- **Verification Token**（可选但推荐）：防止伪造回调请求
- **消息权限**：仅响应 `/` 开头的命令
- **内网部署**：回调 URL 用 `http://IP:6700`，无需公网域名
- **Token 泄露**：在飞书开放平台重置 App Secret 即可

## ❓ 常见问题

**Q: 回调 URL 验证失败？**  
A: 确保服务器 6700 端口对外开放（检查防火墙/安全组）。

**Q: 命令无响应？**  
A: 检查 `FEISHU_APP_ID` / `FEISHU_APP_SECRET` 是否正确配置。

**Q: 安全吗？**  
A: 机器人只响应 `/` 开头命令，不会处理其他消息。App Secret 泄露可在平台重置。
