# V2EX Max Helper — 服务器部署指南

> 适用于 Linux VPS，推荐内存 ≥ 1 GB

---

## 📋 部署概览

| 组件 | 状态 | 说明 |
|---|---|---|
| Mihomo 代理 | ✅ 运行中 | `systemctl status mihomo`，端口 7890 |
| V2EX 签到 | ✅ 已配置 | 每天 08:00 自动签到 |
| 保活心跳 | ✅ 已配置 | 每 6 小时刷新登录态 |
| 自动阅读 | ⚠️ 需 Cookie | 每天 UTC 06:00，需先填入 Cookie |
| Telegram Bot | ⚠️ 需 Token | 常驻进程，需填入 TG_TOKEN |

## 🚀 快速开始（你还需要做的）

### 第 1 步：获取 V2EX Cookie

1. 用浏览器打开 https://www.v2ex.com 并登录
2. 按 F12 打开开发者工具 → Application → Cookies → www.v2ex.com
3. 复制完整的 Cookie 字符串（包含 `A2=...` 等内容）

### 第 2 步：写入 Cookie

SSH 到服务器后：

```bash
# 将下面 <你的Cookie> 替换为实际值
echo "<你的Cookie>" > /root/.v2ex_cookie

# 验证
cat /root/.v2ex_cookie
```

### 第 3 步：立即测试签到

```bash
# 代理仅对 v2ex 脚本进程生效（不影响服务器其他业务）
cd ~/v2ex-max-helper/checkin
node --require ./preload-proxy.js v2ex-checkin.js
```

### 第 4 步（可选）：配置 Telegram Bot

编辑 `/root/.v2ex_env`：

```bash
vi /root/.v2ex_env
```

填入：
```ini
TG_TOKEN=<你的BotToken>
TG_CHAT_ID=<你的ChatID>
```

然后启动 Bot：
```bash
systemctl start v2ex-bot
systemctl status v2ex-bot
```

## 🔧 常用命令速查

### 代理管理

```bash
# 查看代理状态
systemctl status mihomo

# 测试代理是否正常
proxy-test                          # 或: curl --proxy http://127.0.0.1:7890 https://www.v2ex.com

# 查看代理当前选中的节点
curl -s http://127.0.0.1:9090/proxies | python3 -c "import json,sys;d=json.load(sys.stdin);print(d['proxies']['AutoSelect']['now'])"

# 重启代理
systemctl restart mihomo

# 查看代理日志
journalctl -u mihomo -f
```

### V2EX Helper 管理

```bash
# 查看定时器状态
systemctl list-timers 'v2ex-*'

# 手动触发签到
systemctl start v2ex-checkin

# 查看签到日志
journalctl -u v2ex-checkin -n 50

# 手动触发保活
systemctl start v2ex-ping

# 手动运行阅读（干跑模式）
cd ~/v2ex-max-helper/reader && node --require ./preload-proxy.js main.js --dry-run

# 手动运行阅读（限制5篇）
cd ~/v2ex-max-helper/reader && node --require ./preload-proxy.js main.js --limit 5

# 手动运行签到
cd ~/v2ex-max-helper/checkin && node --require ./preload-proxy.js v2ex-checkin.js

# 查看阅读日志
journalctl -u v2ex-reader -n 50

# Bot 状态
systemctl status v2ex-bot
journalctl -u v2ex-bot -n 30
```

## ⏰ 定时任务安排

| 任务 | 定时器 | 说明 |
|---|---|---|
| 每日签到 | `v2ex-checkin.timer` | 每天 08:00 CST ±30分钟随机延迟 |
| 保活心跳 | `v2ex-ping.timer` | 每 6 小时执行（00:15, 06:15, 12:15, 18:15） |
| 自动阅读 | `v2ex-reader.timer` | 每天 UTC 06:00 (±30分钟随机延迟) |

## 🌐 代理架构（进程级局部代理，不影响其他服务）

```
只有 V2EX Helper 进程走代理 ← 其他服务不受影响
┌─────────────────────────────────────────────────────┐
│  服务器其他业务（酒馆 ST :8000 等）                   │
│  → 直连互联网，不走代理                              │
├─────────────────────────────────────────────────────┤
│  v2ex-checkin / v2ex-reader / v2ex-bot (Node.js)   │
│  → --require ./preload-proxy.js                     │
│  → https-proxy-agent 劫持 https.request             │
│  → 所有 V2EX 请求 → 127.0.0.1:7890 → Mihomo → 境外  │
│  → Playwright 浏览器 → proxy: 127.0.0.1:7890         │
├─────────────────────────────────────────────────────┤
│  Mihomo v1.19.4 (仅监听 127.0.0.1:7890)              │
│  → url-test 自动选最快节点（36个代理，每5分钟测速）    │
│  → 订阅1: vmess/ssr  订阅2: anytls/hysteria2/trojan  │
└─────────────────────────────────────────────────────┘
```

## 🛠 故障排查

### 代理不通

```bash
# 检查 mihomo 是否运行
systemctl status mihomo

# 检查端口
ss -tlnp | grep 7890

# 查看节点状态
curl -s http://127.0.0.1:9090/proxies

# 重启代理
systemctl restart mihomo
```

### 签到失败

```bash
# 先测试 Cookie 是否有效
cd ~/v2ex-max-helper/checkin && node --require ./preload-proxy.js v2ex-checkin.js

# 查看详细错误
journalctl -u v2ex-checkin -n 50 --no-pager -l
```

### 更新订阅节点

```bash
# 代理节点会自动从订阅 URL 更新（每 3600 秒）
# 手动强制更新：
curl -s http://127.0.0.1:9090/providers/proxies
```

## 🗑 卸载

```bash
# 停止并禁用所有服务
systemctl disable --now v2ex-checkin.timer v2ex-ping.timer v2ex-reader.timer v2ex-bot.service

# 删除项目
rm -rf ~/v2ex-max-helper

# 删除配置
rm -f /root/.v2ex_cookie /root/.v2ex_env
```
