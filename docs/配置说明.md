# 配置说明

配置通过 **环境变量** 或 **`~/.v2ex_env` 文件** 提供，代码中不含任何个人信息。
环境变量优先级高于 `~/.v2ex_env` 文件。

> `checkin/` 与 `reader/` 都通过 `lib/config.js` 读取同一套配置。环境变量仍然优先，
> `~/.v2ex_env` 只会补充未设置的变量。

## 配置项一览

| 变量名 | 用于 | 读取来源 | 必填 | 说明 |
|--------|------|----------|------|------|
| `TG_TOKEN` | reader（bot/notify）、checkin 推送 fallback | env 文件 / 环境变量 | Bot 必填 | Telegram Bot Token；签到脚本也会读取它 |
| `TG_CHAT_ID` | reader（bot/notify）、checkin 推送 | env 文件 / 环境变量 / 运行时绑定文件 | 二选一 | 唯一授权 Chat ID；直接配置时最稳 |
| `TG_SETUP_CODE` | reader（bot） | env 文件 / 环境变量 | 二选一 | 未配置 `TG_CHAT_ID` 时必填；首次绑定必须发送 `/bind <口令>` |
| `TG_BOT_TOKEN` | checkin 推送 | env 文件 / 环境变量 | 可选 | 旧变量名；设置后优先于 `TG_TOKEN` |
| `BARK_URL` | checkin 推送 | env 文件 / 环境变量 | 可选 | Bark 推送地址，如 `https://api.day.app/你的KEY` |
| `READER_LOG` | reader（bot） | env 文件 / 环境变量 | 可选 | 阅读日志路径，`/debug` 命令读取，默认运行时数据目录下的 `v2ex-reader.log` |
| `V2EX_PROFILE` | 全部 | env 文件 / 环境变量 | 可选 | 单账号默认 `default`；多账号任务必须显式指定。名称仅允许字母、数字、下划线和连字符 |
| `V2EX_PROFILE_LIST` | reader（bot 调度器） | env 文件 / 环境变量 | 可选 | 多账号串行及 Telegram 控制列表，如 `acc1,acc2`；去重后最多 6 个，不能使用仅大小写不同的名称 |
| `PROFILE_TIME_SLOT_HOURS` | reader（bot 调度器） | env 文件 / 环境变量 | 可选 | 每个账号“签到 + 阅读”的完整固定窗口，默认 `4` 小时；账号数乘窗口不得超过 24 小时 |
| `V2EX_DATA_DIR` | 全部 | env 文件 / 环境变量 | 可选 | 运行时数据目录；相对路径统一按仓库根目录解析，Docker / Render 默认 `/app/data` |
| `TZ` | Docker / Render / Node.js | env 文件 / 环境变量 | 可选 | 运行环境时区；Compose 默认 `Asia/Shanghai`，可用任意有效 IANA 时区覆盖，调度与截止时间按本机时区计算 |
| `COOKIE_FILE` | 全部 | env 文件 / 环境变量 | 可选 | Cookie 文件路径。显式设置时优先；否则按 profile：`default`→`~/.v2ex_cookie`，其他→`~/.v2ex_cookie.<profile>` |
| `V2EX_COOKIE` | 全部 | env 文件 / 环境变量 | 可选 | 单账号启动凭证或 `--save-cookie` 输入；先验证后写盘。多账号模式必须同时显式指定列表内的 `V2EX_PROFILE`，不会复制到其他账号 |
| `DB_PATH` | reader（queue） | env 文件 / 环境变量 | 可选 | 单账号运行时可覆盖 SQLite 路径；默认 `queue.db`，非默认 profile 自动使用 `queue.<profile>.db`。Bot 多账号子任务忽略该共享覆盖值 |
| `V2EX_PROXY_ENABLE` | 全部网络请求 | env 文件 / 环境变量 | 可选 | 设为 `1` 才启用代理；默认关闭 |
| `V2EX_PROXY_ALLOW_LAN` | 全部网络请求 | env 文件 / 环境变量 | 可选 | 设为 `1` 时允许 RFC1918 局域网代理 IP；默认仍只允许本机代理 |
| `V2EX_PROXY` | 全部网络请求 | env 文件 / 环境变量 | 可选 | 代理地址；支持 `http://`、`https://`、`socks5://`。HTTP(S) 可带账号密码；浏览器不支持带认证 SOCKS5。代理软件不限定 mihomo，也可以是 xray-core、sing-box、Clash、Surge、gost 等 |
| `FEISHU_ENABLE` | reader/checkin 推送 | env 文件 / 环境变量 | 可选 | 设为 `1` 才启用飞书 Webhook 推送；默认关闭 |
| `FEISHU_WEBHOOK` | reader/checkin 推送 | env 文件 / 环境变量 | 可选 | 飞书自定义机器人 Webhook；仅在 `FEISHU_ENABLE=1` 时使用 |
| `FEISHU_BOT_ENABLE` | reader（feishu-bot） | env 文件 / 环境变量 | 可选 | 设为 `1` 且单独运行 `node reader/feishu-bot.js` 才启动飞书交互 Bot |
| `FEISHU_APP_ID` / `FEISHU_APP_SECRET` | reader（feishu-bot） | env 文件 / 环境变量 | 可选 | 飞书应用凭据；启用交互 Bot 时必需 |
| `FEISHU_VERIFICATION_TOKEN` | reader（feishu-bot） | env 文件 / 环境变量 | 可选 | 飞书回调校验 token；启用交互 Bot 时必需 |
| `FEISHU_CHAT_ID` | reader（feishu-bot） | env 文件 / 环境变量 | 可选 | 唯一授权会话；启用交互 Bot 时必需，其他会话静默忽略 |
| `FEISHU_BOT_PORT` | reader（feishu-bot） | env 文件 / 环境变量 | 可选 | 飞书交互 Bot 回调端口，默认 `6700` |

## 安装脚本变量

这些变量只影响部署脚本，不影响运行中的签到 / 阅读逻辑：

| 变量名 | 说明 |
|--------|------|
| `V2EX_LANG` | 部署脚本语言，支持 `zh` / `en`；也可使用 `--lang zh` / `--lang en` |
| `INSTALL_DIR` | 一键部署目录，默认 `~/v2ex-max-helper` |
| `SKIP_READER` | 设为 `1` 时只安装签到 + 保活，不安装自动阅读模块 |

## `~/.v2ex_env` 示例

```ini
TG_TOKEN=<在此填入你的BotToken>
# TG_CHAT_ID=<在此填入你的Chat_ID>
# TG_SETUP_CODE=<未配置TG_CHAT_ID时必填；首次私聊发送 /bind 口令>
# TG_BOT_TOKEN=<在此填入你的BotToken>
# BARK_URL=https://api.day.app/你的KEY
# READER_LOG=/var/log/v2ex-reader.log
# V2EX_PROFILE_LIST=acc1,acc2
# PROFILE_TIME_SLOT_HOURS=4
# V2EX_PROXY_ENABLE=1
# V2EX_PROXY_ALLOW_LAN=1
# V2EX_PROXY=http://127.0.0.1:7890
# V2EX_PROXY=socks5://127.0.0.1:7890
# FEISHU_ENABLE=1
# FEISHU_WEBHOOK=https://open.feishu.cn/open-apis/bot/v2/hook/你的KEY
# FEISHU_BOT_ENABLE=1
# FEISHU_APP_ID=<飞书应用App ID>
# FEISHU_APP_SECRET=<飞书应用App Secret>
# FEISHU_VERIFICATION_TOKEN=<飞书事件订阅校验token>
# FEISHU_CHAT_ID=<唯一授权飞书会话ID>
# FEISHU_BOT_PORT=6700
```

> 文件格式为 `键=值`，每行一条，键必须是大写字母、数字和下划线。务必 `chmod 600 ~/.v2ex_env`。

> ⚠️ 为避免 Bot 被陌生人抢先绑定，未配置 `TG_CHAT_ID` 且未配置 `TG_SETUP_CODE` 时，Bot 会保持未绑定状态并拒绝所有命令。无口令首次自动绑定已关闭。

> 飞书 Webhook 和交互 Bot 均默认关闭，属于社区讨论阶段能力；如需调整权限模型、命令范围或部署方式，建议提交 issue 或 PR。

> 启用 `V2EX_PROXY_ENABLE=1` 后，代理地址、协议或主机校验失败会直接终止启动，不会回退直连，避免在误配置时暴露真实出口。

> `default` 继续使用旧的 `balance_log.json`、`balance_status.json` 和 `queue.db`；非默认 profile 使用带名称的独立文件。升级前已有的共享余额与队列无法可靠判断所属账号，因此不会复制或自动迁移给其他 profile。

每个 profile 还会生成 `profile_identity[.<profile>].json` 和 `.locks/credential.<profile>.lock`。身份文件只保存随机盐及账号标识哈希，不保存明文用户名；凭证锁用于阻止签到、阅读和 Cookie 导入同时覆盖同一账号。全局 `profile_schedule_state.json` 保存固定时段进度，阅读锁则记录 PID、profile、任务类型和启动时间。

Cookie 导入采用完整候选替换：至少包含 `A2`，验证失败不写盘；同一 profile 检测到账号变化时必须在 Telegram 二次确认，并清理旧 Chromium 登录状态。旧版本升级后首次建立身份绑定时也会重建该 profile 的 Chromium 状态，避免遗留的 localStorage、IndexedDB 等浏览器凭证混入新绑定。系统不会再把新 `A2` 与旧 `PB3_SESSION`、`cf_clearance` 或其他旧字段拼接。

## 阅读脚本可调参数

在 `reader/main.js` 顶部：

| 常量 | 默认 | 含义 |
|------|------|------|
| `MAX_READ_COUNT` | 1000 | 每日阅读篇数上限（安全兜底） |
| `MAX_CHANGE_COUNT` | 2 | 余额变化达到此次数即停止（活跃度奖励通常两轮） |
| `BALANCE_CHECK_INTERVAL` | 约 45~55 | 每读多少篇检查一次余额；未显式设置时会按 profile 产生小幅稳定偏移 |
| `DEADLINE_LOCAL_HOUR` | 14 | 本机时间 14:00 截止，超时强制退出 |

## 阅读节奏随机化

为模拟真人阅读，停留时长采用**偏态分布**（多数偏短、偶尔长读），并在帖子之间插入随机间隔，
停留期间还会随机滚动页面。以下参数可用环境变量覆盖（单位毫秒）：

| 环境变量 | 默认 | 含义 |
|----------|------|------|
| `READ_DWELL_MIN` | 8000 | 单篇停留最短时长 |
| `READ_DWELL_MAX` | 22000 | 单篇常规停留最长时长 |
| `READ_DWELL_LONG` | 45000 | 偶尔「长读」的上限 |
| `READ_LONG_CHANCE` | 0.15 | 触发长读的概率（0~1） |
| `READ_HUMAN_GAP_MIN` | 3000 | 帖子之间拟人切换间隔最短 |
| `READ_HUMAN_GAP_MAX` | 9000 | 帖子之间拟人切换间隔最长 |
| `READ_MEMORY_SETTLE_MS` | 5000 | 每篇读完后给 Chromium 回收资源的固定等待 |
| `READ_GAP_MIN` | 兼容旧变量 | 旧版帖子间隔最短；未设置 `READ_HUMAN_GAP_MIN` 时作为 fallback |
| `READ_GAP_MAX` | 兼容旧变量 | 旧版帖子间隔最长；未设置 `READ_HUMAN_GAP_MAX` 时作为 fallback |

默认配置下，单篇停留中位数约 13 秒、均值约 16 秒，约 15% 概率拉长到 22~45 秒，
帖子间额外拟人间隔约 3~9 秒，并额外等待约 5 秒释放浏览器资源。想更保守（更慢更像人）可调大这些值。

未显式设置 `READ_HUMAN_GAP_MIN`（或旧变量 `READ_GAP_MIN`）和 `READ_MEMORY_SETTLE_MS` 时，程序会保证“最短拟人间隔 + 内存等待”不少于 8000ms。显式设置其中任一最短值时视为高级覆盖，允许低于该默认下限。

> 🧠 v1.3.2 起，拟人间隔和内存释放等待已经拆开：`READ_HUMAN_GAP_*` 只负责阅读节奏，
> `READ_MEMORY_SETTLE_MS` 只负责给 Chromium 留出资源回收窗口，避免两个目的混在同一组参数里。

未显式设置 `READ_*` 时，阅读停留、长读概率、帖子间隔和余额检查间隔会基于 `V2EX_PROFILE`
产生小幅、确定性的差异；同一 profile 每次保持一致，不会每次启动随机漂移。
