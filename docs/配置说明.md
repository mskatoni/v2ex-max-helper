# 配置说明

配置通过 **环境变量** 或 **`~/.v2ex_env` 文件** 提供，代码中不含任何个人信息。
环境变量优先级高于 `~/.v2ex_env` 文件。

> ⚠️ **重要区别**：只有 **reader 模块**（`bot.js` / `notify.js`）会自动读取 `~/.v2ex_env` 文件；
> **checkin 模块**（`v2ex-checkin.js`）只读取**进程环境变量**，不读 `~/.v2ex_env`。
> 因此给签到配置 Telegram/Bark 推送时，需在命令或 systemd service 的 `Environment=` 里直接传入，
> 例如：`TG_BOT_TOKEN="xxx" TG_CHAT_ID="xxx" node v2ex-checkin.js`。

## 配置项一览

| 变量名 | 用于 | 读取来源 | 必填 | 说明 |
|--------|------|----------|------|------|
| `TG_TOKEN` | reader（bot/notify） | env 文件 / 环境变量 | Bot 必填 | Telegram Bot Token |
| `TG_CHAT_ID` | reader（bot/notify）、checkin 推送 | env 文件（reader）/ 环境变量 | Bot 必填 | 唯一授权 Chat ID，Bot 硬锁只响应它 |
| `TG_BOT_TOKEN` | checkin 推送 | **仅环境变量** | 可选 | 签到脚本推送用的 Bot Token |
| `BARK_URL` | checkin 推送 | **仅环境变量** | 可选 | Bark 推送地址，如 `https://api.day.app/你的KEY` |
| `READER_LOG` | reader（bot） | env 文件 / 环境变量 | 可选 | 阅读日志路径，`/debug` 命令读取，默认 `/var/log/v2ex-reader.log` |
| `V2EX_PROFILE` | 全部 | 环境变量 | 可选 | 多账号 profile 名，默认 `default`。隔离 Cookie 文件 / 浏览器数据 / 指纹，详见 [多账号与指纹隔离.md](多账号与指纹隔离.md) |
| `COOKIE_FILE` | 全部 | 环境变量 | 可选 | Cookie 文件路径。显式设置时优先；否则按 profile：`default`→`~/.v2ex_cookie`，其他→`~/.v2ex_cookie.<profile>` |
| `V2EX_COOKIE` | checkin（--save-cookie） | 环境变量 | — | 仅在保存 Cookie 时临时传入 |
| `DB_PATH` | reader（queue） | 环境变量 | 可选 | SQLite 数据库路径，默认 `reader/data/queue.db` |

## `~/.v2ex_env` 示例

```ini
TG_TOKEN=<在此填入你的BotToken>
TG_CHAT_ID=<在此填入你的Chat_ID>
TG_BOT_TOKEN=<在此填入你的BotToken>
# BARK_URL=https://api.day.app/你的KEY
# READER_LOG=/var/log/v2ex-reader.log
```

> 文件格式为 `键=值`，每行一条，键必须是大写字母和下划线。务必 `chmod 600 ~/.v2ex_env`。

## 阅读脚本可调参数

在 `reader/main.js` 顶部：

| 常量 | 默认 | 含义 |
|------|------|------|
| `MAX_READ_COUNT` | 1000 | 每日阅读篇数上限（安全兜底） |
| `MAX_CHANGE_COUNT` | 2 | 余额变化达到此次数即停止（活跃度奖励通常两轮） |
| `BALANCE_CHECK_INTERVAL` | 50 | 每读多少篇检查一次余额 |
| `DEADLINE_UTC_HOUR` | 6 | 截止时间（UTC 06:00 = 北京 14:00），超时强制退出 |

## 阅读节奏随机化

为模拟真人阅读，停留时长采用**偏态分布**（多数偏短、偶尔长读），并在帖子之间插入随机间隔，
停留期间还会随机滚动页面。以下参数可用环境变量覆盖（单位毫秒）：

| 环境变量 | 默认 | 含义 |
|----------|------|------|
| `READ_DWELL_MIN` | 8000 | 单篇停留最短时长 |
| `READ_DWELL_MAX` | 22000 | 单篇常规停留最长时长 |
| `READ_DWELL_LONG` | 45000 | 偶尔「长读」的上限 |
| `READ_LONG_CHANCE` | 0.15 | 触发长读的概率（0~1） |
| `READ_GAP_MIN` | 8000 | 帖子之间间隔最短（**代码强制下限 8 秒**，设更小也会被抬到 8 秒） |
| `READ_GAP_MAX` | 15000 | 帖子之间间隔最长 |

默认配置下，单篇停留中位数约 13 秒、均值约 16 秒，约 15% 概率拉长到 22~45 秒，
帖子间额外间隔 8~15 秒。想更保守（更慢更像人）可调大这些值。

> 🧠 **帖子间隔为何不低于 8 秒**：除了更拟人，这段间隔也给 Chromium 留出垃圾回收时间，
> 让它在加载下一页前先释放上一页内存。低内存机器（如 1GB）翻页过快容易在内存峰值叠加时被
> OOM Killer 杀掉，因此代码层把最短间隔强制锁定为 8 秒。

实现位于 `reader/browser.js` 的 `readPost()` 与 `randomDwellMs()` / `dwellWithScroll()`。
