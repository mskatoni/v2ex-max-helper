## 代理组说明

| 组名 | 类型 | 策略 | 说明 |
|---|---|---|---|
| `AutoSelect` | url-test | 每 5 分钟测速，选最快节点 | **默认路由组**，36 个节点自动竞争 |
| `Fallback` | fallback | 按顺序尝试，选第一个可用 | 备用故障转移 |
| `GLOBAL` | selector | 手动选择 | 可手动指定走哪个节点或直连 |

## 多订阅源

支持配置多个代理订阅源，Mihomo 会自动合并并去重所有节点。在 `/etc/mihomo/config.yaml` 的 `proxy-providers` 中添加即可：

```yaml
proxy-providers:
  sub1:
    type: http
    url: "https://你的订阅地址1"
    interval: 3600
    path: ./provider-sub1.yaml
    health-check:
      enable: true
      url: https://www.gstatic.com/generate_204
      interval: 300

  sub2:
    type: http
    url: "https://你的订阅地址2"
    interval: 3600
    path: ./provider-sub2.yaml
    health-check:
      enable: true
      url: https://www.gstatic.com/generate_204
      interval: 300
```

### 自动切换机制

1. `url-test` 组定时（每 300 秒）对每个节点进行测速
2. 测速 URL: `https://cp.cloudflare.com/generate_204`
3. 自动选择延迟最低的可用节点
4. 如果当前节点失效，`Fallback` 组自动切换到下一个可用节点

## 手动操作

### 查看当前使用的节点
```bash
curl -s http://127.0.0.1:9090/proxies | python3 -c "
import json,sys
d=json.load(sys.stdin)
print('AutoSelect →', d['proxies']['AutoSelect']['now'])
print('Fallback  →', d['proxies']['Fallback']['now'])
"
```

### 测试代理速度
```bash
# 通过代理访问 V2EX
curl -s --proxy http://127.0.0.1:7890 https://www.v2ex.com -o /dev/null -w "HTTP:%{http_code} Time:%{time_total}s\n"

# 通过代理访问 Google
curl -s --proxy http://127.0.0.1:7890 https://www.google.com -o /dev/null -w "HTTP:%{http_code} Time:%{time_total}s\n"
```

### 查看所有节点及其延迟
```bash
curl -s http://127.0.0.1:9090/proxies/AutoSelect | python3 -c "
import json,sys
d=json.load(sys.stdin)
for name,info in d.get('all',{}).items():
    delay = info.get('history',[{}])[-1].get('delay',0) if info.get('history') else 0
    print(f'{name:40s} {delay}ms')
"
```

## 添加/更换订阅

编辑 `/etc/mihomo/config.yaml`，在 `proxy-providers` 下添加新的提供者，然后重启：

```bash
systemctl restart mihomo
```
