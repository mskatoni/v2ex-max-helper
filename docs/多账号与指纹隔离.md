# 多账号与浏览器指纹隔离

本项目支持同时管理多个 V2EX 账号，并为每个账号提供**独立、确定性的浏览器指纹**，
降低多账号被关联、被风控的概率。

> ⚠️ 多账号自动化属于较激进的玩法，V2EX 明确禁止刷活跃度。请自行评估封号风险，
> 控制账号数量与频率，**后果自负**。

---

## 一、核心概念：Profile

通过环境变量 `V2EX_PROFILE` 指定当前操作的账号。每个 profile 之间**完全隔离**：

| 资源 | `default`（不设 profile） | `<profile>`（如 `acc1`） |
|------|--------------------------|--------------------------|
| Cookie 文件 | `~/.v2ex_cookie` | `~/.v2ex_cookie.<profile>` |
| Chrome 用户数据目录 | `reader/data/chrome-profile/default` | `reader/data/chrome-profile/<profile>` |
| 浏览器指纹 | 由 `default` 种子生成 | 由 `<profile>` 种子生成 |

> 显式设置 `COOKIE_FILE` 环境变量时，会覆盖上面的 Cookie 文件路径规则。

---

## 二、指纹隔离做了什么

`reader/fingerprint.js` 以 **profile 名为种子**，确定性地生成一套指纹，保证：

- **同一账号每次启动指纹完全一致**（避免自己跟自己对不上而更可疑）；
- **不同账号之间指纹互相隔离**（UA、视口、时区、语言、硬件参数、WebGL 各不相同）。

隔离维度：

| 维度 | 说明 |
|------|------|
| User-Agent | 近期主流 Chrome 大版本（122~126）+ Windows / macOS 平台 |
| 视口尺寸 | 常见桌面分辨率（1280×720 ~ 1920×1080） |
| 时区 | 美区常见时区：New_York、Los_Angeles、Chicago、Denver、Phoenix |
| 语言 | 以 en-US 为主（美区环境），部分含 zh-CN 辅助，含 `Accept-Language` |
| navigator.platform | Win32 / MacIntel |
| hardwareConcurrency | 4 / 8 / 12 / 16 |
| deviceMemory | 4 / 8 (GB) |
| WebGL vendor/renderer | Intel / NVIDIA / AMD / Apple 常见显卡 |
| navigator.webdriver | 隐藏（伪装为非自动化） |

所有取值都限定在**常见、低风险**的真实组合内，避免生成罕见参数反而更显眼。

> 💡 **零额外开销**：指纹隔离仅在启动时做一次哈希计算（微秒级），不引入任何新依赖，
> 也**不会多开浏览器**。单账号或多账号串行运行时，资源占用与未启用指纹隔离时**完全相同**。

## 三、使用示例

### 1. 为每个账号保存 Cookie

```bash
cd checkin

# 账号 1（默认 profile）
V2EX_COOKIE="账号1的cookie" node v2ex-checkin.js --save-cookie
# → 保存到 ~/.v2ex_cookie

# 账号 2
V2EX_PROFILE=acc2 V2EX_COOKIE="账号2的cookie" node v2ex-checkin.js --save-cookie
# → 保存到 ~/.v2ex_cookie.acc2

# 账号 3
V2EX_PROFILE=acc3 V2EX_COOKIE="账号3的cookie" node v2ex-checkin.js --save-cookie
# → 保存到 ~/.v2ex_cookie.acc3
```

### 2. 多账号签到

```bash
cd checkin
node v2ex-checkin.js                 # 账号1
V2EX_PROFILE=acc2 node v2ex-checkin.js   # 账号2
V2EX_PROFILE=acc3 node v2ex-checkin.js   # 账号3
```

### 3. 多账号自动阅读（每个账号独立指纹）

```bash
cd reader
node main.js                         # 账号1，指纹 A
V2EX_PROFILE=acc2 node main.js       # 账号2，指纹 B
V2EX_PROFILE=acc3 node main.js       # 账号3，指纹 C
```

> 启动时日志会打印当前 profile 与指纹摘要，便于核对。

## 四、重要建议

- **强烈建议错开时间**运行各账号，不要并发同时跑，行为更自然，也省内存（每个阅读实例都会起一个 Chromium）。
- **建议每个账号配独立出口 IP**（不同代理/VPS）。指纹隔离能降低浏览器层关联，
  但同 IP 多账号仍是最容易被关联的特征，指纹隔离无法解决 IP 维度的问题。
- 账号越多，封号风险越高。请克制。

### 多账号串行运行示例脚本

```bash
#!/usr/bin/env bash
# run-all.sh —— 依次为多个账号签到（错峰可加 sleep）
set -e
for p in default acc2 acc3; do
  echo "===== profile: $p ====="
  V2EX_PROFILE="$p" node checkin/v2ex-checkin.js || true
  sleep $((RANDOM % 120 + 30))   # 随机间隔 30~150 秒，行为更自然
done
```

## 五、配合定时任务

### 推荐：systemd timer（每个账号独立单元）

用一键脚本为每个 profile 生成独立的 timer，并用不同时间错峰：

```bash
sudo bash scripts/install-systemd.sh                  # 账号1（default）
sudo bash scripts/install-systemd.sh --profile acc2   # 账号2
sudo bash scripts/install-systemd.sh --profile acc3   # 账号3
```

> 脚本运行时会**交互式询问**各任务的执行时间，给不同账号填不同时间即可实现错峰。

每个 profile 会生成 `v2ex-checkin-<profile>`、`v2ex-ping-<profile>`、`v2ex-reader-<profile>` 等单元，
互不干扰。`RandomizedDelaySec` 还会在你设定的时间上再叠加随机抖动，错峰更自然。查看：

```bash
systemctl list-timers 'v2ex-*'
```

### 回退：crontab（无 systemd 时）

```cron
# 账号1 09:10、账号2 09:25、账号3 09:40（北京），错峰签到
10 1 * * * /usr/bin/node /path/to/checkin/v2ex-checkin.js >> /var/log/v2ex.log 2>&1
25 1 * * * V2EX_PROFILE=acc2 /usr/bin/node /path/to/checkin/v2ex-checkin.js >> /var/log/v2ex.log 2>&1
40 1 * * * V2EX_PROFILE=acc3 /usr/bin/node /path/to/checkin/v2ex-checkin.js >> /var/log/v2ex.log 2>&1
```
