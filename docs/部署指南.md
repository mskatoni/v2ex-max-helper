# 部署指南（手动）

本文以一台全新的 Linux VPS（Debian/Ubuntu）为例，介绍从零**手动**部署 V2EX Max Helper 的完整流程，
便于理解每一步。**只想快速跑起来**的话，直接用一键脚本即可（root 执行）：

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/mskatoni/v2ex-max-helper/main/scripts/install.sh)
```

更新已有安装：

```bash
bash ~/v2ex-max-helper/scripts/install.sh --update
```

> 💡 **不想手敲命令？** 也可让 AI 助手代为部署，详见 [Agent辅助部署.md](Agent辅助部署.md)。
> 推荐 **Claude Pro**（付费）或 **Antigravity / Google Antigravity**（免费，
> 官网 <https://antigravity.google> ，Google 账号 Free 计划即可用 Claude Opus 4.6 等模型）。
> ⚠️ 启动 Agent 时请用官方直连 API、不要经过第三方「中转站」，并把 Cookie、Token 留到最后由你本人手动写入。

## 一、准备环境

### 1. 安装 Node.js（18+）

```bash
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs
node -v   # 确认 >= 18
```

### 2. 下载项目

```bash
cd ~
# 方式 A：下载 zip
wget https://github.com/mskatoni/v2ex-max-helper/archive/refs/heads/main.zip
unzip main.zip
mv v2ex-max-helper-main v2ex-max-helper
cd v2ex-max-helper

# 方式 B：git clone
# git clone https://github.com/mskatoni/v2ex-max-helper.git
# cd v2ex-max-helper
```

## 二、配置敏感信息

```bash
cp .v2ex_env.example ~/.v2ex_env
nano ~/.v2ex_env        # 填入 TG_TOKEN / TG_CHAT_ID 等
chmod 600 ~/.v2ex_env
```

- `TG_TOKEN`：向 [@BotFather](https://t.me/BotFather) 创建 Bot 获取。
- `TG_CHAT_ID`：向 [@userinfobot](https://t.me/userinfobot) 发消息获取你的数字 ID。

详见 [配置说明.md](配置说明.md)。

## 三、保存 V2EX Cookie

1. 浏览器登录 V2EX，按 F12 打开开发者工具。
2. Network 标签页刷新页面，点任意请求，复制 Request Headers 里的整段 `Cookie`。
3. 在 VPS 上执行：

```bash
cd ~/v2ex-max-helper/checkin
V2EX_COOKIE="此处粘贴你的完整cookie" node v2ex-checkin.js --save-cookie
```

成功后会生成 `~/.v2ex_cookie`（权限 600）。签到和阅读模块共用此文件。

## 四、签到模块

```bash
cd ~/v2ex-max-helper/checkin
node v2ex-checkin.js          # 立即签到测试
node v2ex-checkin.js --ping   # 保活心跳测试
```

## 五、阅读模块

```bash
cd ~/v2ex-max-helper/reader
npm install                          # 安装 playwright + sql.js
npx playwright install chromium      # 安装浏览器内核（首次必需）

node main.js --dry-run               # 干跑，验证流程
node main.js                         # 正式运行
```

> 注意：`browser.js` 默认以 `headless: false`（有头模式）启动以降低风控。无头服务器需配合 `xvfb-run`：
> ```bash
> sudo apt-get install -y xvfb
> xvfb-run -a node main.js
> ```

## 六、Telegram Bot（常驻）

推荐用 `systemd` 守护，示例 `/etc/systemd/system/v2ex-bot.service`：

```ini
[Unit]
Description=V2EX Telegram Bot
After=network.target

[Service]
Type=simple
WorkingDirectory=/home/youruser/v2ex-max-helper/reader
ExecStart=/usr/bin/node bot.js
Restart=always
User=youruser
Environment=HOME=/home/youruser

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now v2ex-bot
sudo systemctl status v2ex-bot
```

> 一键部署脚本（`install.sh`）在 Step 7/8 会交互式询问是否安装 Bot。若当时跳过，可稍后执行：
> ```bash
> sudo bash scripts/install-systemd.sh --bot
> ```

## 七、定时任务（推荐 systemd timer）

> 本节针对**定时执行**的任务（签到 / 保活 / 阅读）。Telegram Bot 是常驻进程，
> 已在[第六节](#六telegram-bot常驻)用 systemd **service** 配置，不在此列。
>
> **推荐用 systemd timer 而非 crontab。** 相比 cron，timer 有三个对本项目很关键的优势：
> - **`Persistent=true`**：VPS 关机/重启错过的任务，开机后**自动补跑**（避免漏签）。
> - **`RandomizedDelaySec`**：内置随机延迟，天然**错峰**，降低风控特征。
> - **日志统一进 journald**：`journalctl -u v2ex-checkin` 直接看历史与失败原因。
>
> 无 systemd 的老系统（如部分 Alpine/OpenRC）可回退到文末的 **「附：crontab（兼容回退方案）」**。

### 方式 A：一键安装脚本（最省事）

仓库已提供 [`scripts/install-systemd.sh`](../scripts/install-systemd.sh)，自动生成并启用
签到、保活、阅读三个 timer（以及可选的 Bot service）。

```bash
cd v2ex-max-helper
sudo bash scripts/install-systemd.sh
```

脚本会交互式询问运行用户、项目路径、是否启用阅读/ Bot，并自动检测 `node`、`xvfb-run` 路径。
完成后用以下命令查看：

```bash
systemctl list-timers 'v2ex-*'        # 查看所有定时器与下次触发时间
journalctl -u v2ex-checkin -n 50      # 查看签到日志
```

多账号：脚本支持 `--profile <name>` 为指定账号生成独立单元（如 `v2ex-checkin-acc2`）。

```bash
sudo bash scripts/install-systemd.sh --profile acc2
```

卸载：

```bash
sudo bash scripts/install-systemd.sh --uninstall            # 卸载默认账号
sudo bash scripts/install-systemd.sh --uninstall --profile acc2
```

### 方式 B：手动配置 systemd 单元

签到 / 保活示例默认按服务器本地时区解析 `OnCalendar`。自动阅读建议使用显式 UTC 时间，
默认 `01:15 UTC` 启动，和 `reader/main.js` 的 `06:00 UTC` 截止窗口对齐。

**1) 签到 service + timer**

`/etc/systemd/system/v2ex-checkin.service`：

```ini
[Unit]
Description=V2EX 每日签到
After=network-online.target
Wants=network-online.target

[Service]
Type=oneshot
User=youruser
Environment=HOME=/home/youruser
# 签到脚本不读 ~/.v2ex_env；要开启 Telegram/Bark 推送，请在此追加（可选）：
# Environment=TG_BOT_TOKEN=你的BotToken
# Environment=TG_CHAT_ID=你的ChatID
# Environment=BARK_URL=https://api.day.app/你的KEY
WorkingDirectory=/home/youruser/v2ex-max-helper/checkin
ExecStart=/usr/bin/node v2ex-checkin.js
```

`/etc/systemd/system/v2ex-checkin.timer`：

```ini
[Unit]
Description=V2EX 每日签到定时器

[Timer]
OnCalendar=*-*-* 09:10:00
RandomizedDelaySec=600
Persistent=true

[Install]
WantedBy=timers.target
```

**2) 保活 service + timer**（每 6 小时）

`/etc/systemd/system/v2ex-ping.service`：

```ini
[Unit]
Description=V2EX 保活心跳
After=network-online.target
Wants=network-online.target

[Service]
Type=oneshot
User=youruser
Environment=HOME=/home/youruser
WorkingDirectory=/home/youruser/v2ex-max-helper/checkin
ExecStart=/usr/bin/node v2ex-checkin.js --ping
```

`/etc/systemd/system/v2ex-ping.timer`：

```ini
[Unit]
Description=V2EX 保活定时器

[Timer]
OnCalendar=*-*-* 00/6:00:00
RandomizedDelaySec=300
Persistent=true

[Install]
WantedBy=timers.target
```

**3) 自动阅读 service + timer**

`/etc/systemd/system/v2ex-reader.service`：

```ini
[Unit]
Description=V2EX 自动阅读
After=network-online.target
Wants=network-online.target

[Service]
Type=oneshot
User=youruser
Environment=HOME=/home/youruser
WorkingDirectory=/home/youruser/v2ex-max-helper/reader
# 无头服务器用 xvfb-run；有头机器可去掉 xvfb-run 前缀
ExecStart=/usr/bin/xvfb-run -a /usr/bin/node main.js
TimeoutStartSec=6h
```

`/etc/systemd/system/v2ex-reader.timer`：

```ini
[Unit]
Description=V2EX 自动阅读定时器

[Timer]
OnCalendar=*-*-* 01:15:00 UTC
RandomizedDelaySec=900
Persistent=true

[Install]
WantedBy=timers.target
```

**启用：**

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now v2ex-checkin.timer v2ex-ping.timer v2ex-reader.timer
sudo systemctl list-timers 'v2ex-*'
```

> 💡 不想跑阅读模块（只签到+保活）就别启用 `v2ex-reader.timer`。

## 附：crontab（兼容回退方案）

仅在无 systemd 的环境使用。

```bash
crontab -e
```

```cron
# 时间为 UTC（北京时间 = UTC + 8）
10 1 * * * /usr/bin/node /home/youruser/v2ex-max-helper/checkin/v2ex-checkin.js >> /var/log/v2ex.log 2>&1
0 0,6,12,18 * * * /usr/bin/node /home/youruser/v2ex-max-helper/checkin/v2ex-checkin.js --ping >> /var/log/v2ex.log 2>&1
15 1 * * * cd /home/youruser/v2ex-max-helper/reader && /usr/bin/xvfb-run -a node main.js >> /var/log/v2ex-reader.log 2>&1
```

完成。遇到问题请看 [常见问题.md](常见问题.md)。
